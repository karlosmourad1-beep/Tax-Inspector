Right now it looks ugly for a few specific reasons, not because the concept is bad:

Too much empty space
The desk area is huge, while the important things are scattered far apart.
Weak visual hierarchy
The player’s eye does not know where to look first. The queue, desk, directives, lamp, and buttons all compete equally.
Everything is on one flat plane
There is not enough separation between:
background
gameplay surface
documents
controls
side info panel
The orange grid overlay is overpowering
The aesthetic is cool in theory, but it is so strong that it makes text harder to read and makes the whole game feel noisy.
The document workflow is unclear
In a Papers Please style game, the fun comes from a very readable work loop:
call person
inspect docs
compare values
stamp decision

Your screen should make that loop obvious.

What to change first
1. Split the screen into 3 strong zones

A much better layout would be:

Top bar
Day, money, strikes, quota
Center gameplay area
Left: citizen + dialogue
Middle: desk + documents
Right: rules / checklist
Bottom action bar
Approve / Reject / Flag / Next Citizen

That alone will improve the game a lot.

Recommended layout

Think of it like this:

---------------------------------------------------------
TOP BAR: Title | Day | Money | Mistakes | Quota
---------------------------------------------------------
LEFT PANEL      | DESK / DOCUMENT AREA      | RIGHT PANEL
Citizen sprite  | draggable papers          | rules/checklist
Dialogue box    | compare values            | suspicious clues
Queue preview   | stamp / calculator        | current objective
---------------------------------------------------------
BOTTOM ACTION BAR: Approve | Reject | Audit | Next
---------------------------------------------------------
What should visually happen
Background

Keep the regime / ministry vibe, but make the background quieter:

near-black brown background
faint grid at 10 to 15 percent opacity
subtle vignette
maybe one accent glow, not everywhere
Desk

Make the center desk the star of the screen:

large wood or dark matte panel
documents sit on top of it
soft shadow under papers
slight rotation on papers for realism
Right panel

Your “Ministry Directives” should look like a pinned briefing panel:

fixed width
stronger box styling
readable text
current rule highlighted in yellow
completed checks with muted green
Buttons

Current buttons are too big and too harsh. Make them cleaner:

same height
aligned in a row
small icon + label
hover feedback
disable state when unavailable
Visual style direction

You should keep the retro-authoritarian style, but simplify it.

Better palette

Use fewer colors:

background: #120d0a
panel: #1b1410
border: #6f4b1f
accent amber: #e0a11b
muted amber: #9c6b12
success green: #3fa35c
reject red: #b4473f
text: #f3dfb2

Right now the orange is everywhere. That is the biggest style problem.

Typography

The title font is fine for branding, but not for everything.

Use:

a decorative font only for logo/title
a clean monospace or terminal-style font for rules and documents
a readable UI font for buttons and labels

A common mistake is forcing one stylized font onto the whole screen.

Specific design rules to follow
Use 8px or 12px spacing consistently
Keep all interactive controls aligned to a grid
Limit yourself to 1 main accent color
Use borders only where they separate function
Add shadows to papers, not to everything
Make text larger than you think you need
Do not center everything; game UIs usually work better with anchored panels
A strong CSS direction

If your app is React or plain HTML/CSS, I would structure the main gameplay screen with CSS Grid.

Example:

.game-shell {
  display: grid;
  grid-template-rows: 56px 1fr 72px;
  height: 100vh;
  background: #120d0a;
  color: #f3dfb2;
}

.top-bar {
  display: flex;
  align-items: center;

  background: #0e0a08;
}

.main-area {
