Revamp the event newspaper system, add recurring visual characters with disguises, and implement a lightweight story structure with recurring character arcs.

==================================================
PART 1 - FIX THE NEWSPAPER EVENT SCREEN
==================================================

Problem:
The current "newspaper" does not look like a real newspaper. It looks like a centered title card or popup. It does not feel eventful, dramatic, or grounded in the world.

Goal:
On major event days, show an actual newspaper front page that spins or slides onto the screen, clearly showing the world event in a dramatic but readable way.

Requirements:

1. Make it look like a real newspaper front page
- Use an off-white paper background
- Add a masthead at the top (example: "MINISTRY TIMES")
- Add a large bold headline
- Add one short subheadline
- Add 1–2 small boxed sections or columns so it feels like a printed front page
- Optional: add a fake black-and-white image box or chart box

2. Use large readable headline text
Examples:
- MARKET PANIC
- MINISTRY CUTS WAGES
- TAX CRACKDOWN EXPANDS
- BANK RUN IN CAPITAL

3. Keep text minimal
Do NOT use long paragraphs.
Use:
- one main headline
- one short subheadline
- one short effect line

Example:
MARKET PANIC
Emergency decree issued overnight
WAGES -30% TODAY

4. Animate the newspaper
- spin or slide the newspaper into view
- hold for 2–4 seconds
- then transition into the day-start screen

5. Event days only
Do not use the newspaper every day.
Use it only on important days:
- market crash
- crackdown
- ministry scandal
- major policy shift
- riot / panic / financial emergency

6. After the newspaper, show a simple day-start screen
Example:
DAY 3
WAGES -30%
PENALTIES HIGHER
[ START SHIFT ]

Important:
The newspaper is for world flavor and impact.
The day-start screen is for gameplay clarity.

==================================================
PART 2 - ADD VISUAL RECURRING CHARACTERS
==================================================

Goal:
Make the player recognize recurring people visually, without needing full-body sprites.

Implementation:
Use a simple ID-photo system inside the ID card.

Each recurring character should have:
- a base face
- a stable silhouette / face shape
- optional accessories for later days

Possible accessories:
- glasses
- silly glasses
- hat
- fake mustache
- hairstyle variation
- bandage
- beard stubble

Important:
The same person should remain recognizable even with minor changes.

Examples:
- same face + glasses
- same face + hat
- same face + mustache

This should create the feeling:
"Wait, this is that same guy again."

Use this especially for:
- recurring citizens
- fraudsters trying mild disguises
- comedic repeat appearances

Do not make portraits overly realistic.
Use simple stylized ID-photo portraits that are readable and memorable.

==================================================
PART 3 - STORY SYSTEM OVERVIEW
==================================================

Goal:
Turn the game from a pure document-checking system into a story game with recurring people, consequences, and simple branching outcomes.

Rules:
- Keep dialogue short
- No long paragraphs
- Use recurring names, repeated faces, and changing outcomes
- Show consequences later through returns, absences, day events, or small messages

The story should feel like:
- a bureaucratic world under pressure
- citizens returning with history
- the player slowly realizing their decisions affect people

==================================================
PART 5 - STORY CONSEQUENCE RULES
==================================================

Implement simple persistent consequences.

Track for each recurring character:
- times seen
- times approved
- times rejected
- whether frozen
- whether resolved
- whether removed from future appearances

Use these states to change:
- dialogue
- disguises
- whether they come back
- whether later messages appear
- which ending scenes become available

Important:
Do not use long exposition.
Use very short follow-up lines such as:
- "He did not return."
- "Benefits restored."
- "Fraud ring uncovered."
- "Evictions proceed."

Add a few lightweight ending paths based on player behavior.

Possible ending directions:

1. STRICT LOYALIST
- many rejections
- many corporate approvals
- family survives
- citizens suffer

2. COMPASSIONATE CLERK
- more approvals for struggling citizens
- lower money
- better human outcomes

3. FRAUD HUNTER
- catches Adrian and similar cases
- strong ministry reputation
- harsher world tone

4. COLLAPSE / FAILURE
- too much debt, too many penalties, or family breakdown

These do not need to be complex cutscenes.
They can be 1–3 ending panels with images + short lines.

==================================================
PART 7 - IMPLEMENTATION STYLE
==================================================

Keep everything light and readable.

Do:
- short dialogue
- recurring names
- recurring faces
- minor visual disguise changes
- simple state tracking
- short consequence messages

Do not:
- use long dialogue paragraphs
- block the whole screen repeatedly
- overcomplicate story logic

Goal:
Make the player remember people, recognize repeat faces, and feel like their choices changed the world.
